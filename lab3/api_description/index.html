Описание API

/lab3

lab3/reader

lab3/reader/books/\<int:reader_pk>\ Методы: GET\ Описание: Возвращает все книги читателя

lab3/reader/outdated\ Методы: GET\ Описание: Возвращает всех читателей, с просроценными книгами

lab3/reader/rare_books\ Методы: GET\ Описание: Возвращает всех читателей, у которых есть книга, количество экземпляров которой меньше 2-х

lab3/reader/books/book_month_ago\ Методы: GET\ Описание: Возвращает всех читателей, которые взяли книгу более месяца назад

lab3/reader/list\ Методы: GET\ Описание: Возвращает список читателей

lab3/reader/create\ Методы: POST\ Описание: Позволяет создать читателя\ Данные: Все поля читателя

{
    "reader_number": <str>,
    "registration_date": <str> in format of yyyy-mm-dd,
    "active": <bool>,
    "first_name": <str>,
    "last_name": <str>,
    "passport_number": <str>,
    "birth_date": <str> in format of yyyy-mm-dd,
    "address": <str>,
    "mobile_number": <str>
}

{
    "reader_number": "211232",
    "registration_date": "2024-03-07",
    "active": true,
    "first_name": "Anton",
    "last_name": "Votykov",
    "passport_number": "2348172389",
    "birth_date": "2010-01-08",
    "address": "SPB, krasilnikov,  10",
    "mobile_number": "+78005553535"
}

lab3/reader/add_book\ Методы: POST\ Описание: Позволяет дать книгу читателю\ Данные:

{
    "start_date": <date>,
    "end_date": <date> nullabble,
    "book_instance": <int> pk of book instance,
    "reader": <int> pk of reader
}

lab3/statistics

lab3/statistics/education\ Методы: GET\ Описание: Возвращает распределение постетителей библиотеки по уровню образования

lab3/statistics/age\ Методы: GET\ Описание: Возвращает распределение посетителей библиотеки по возрасту

lab3/statistics/room/\<int:pk>\ Методы: GET\ Описание: Возвращает статистику читательского зала\ GET-параметры: * date_after - дата, с которой начинается подсчет статистики * date_before - дата, до которой идет идет подсчет статистики\

Пример запроса: \ /lab3/statistics/room/1?date_after=2023-01-01&date_before=2024-01-01

lab3/statistics/room/list\ Методы: GET\ Описание: Возвращает статистику обоих залов в виде списка GET-параметры: * date_after - дата, с которой начинается подсчет статистики * date_before - дата, до которой идет идет подсчет статистики\

Пример запроса: \ /lab3/statistics/room/list?date_after=2023-01-01&date_before=2024-01-01

lab3/statistics/library\ Методы: GET\ Описание: Возвращает статистику библиотеки GET-параметры: * date_after - дата, с которой начинается подсчет статистики * date_before - дата, до которой идет идет подсчет статистики\

Пример запроса: \ /lab3/statistics/library?date_after=2023-01-01&date_before=2024-01-01

lab3/book_instance

lab3/book_instance/list\ Методы: GET\ Описание: Возвращает список экземпляров книг

lab3/book_instance/\<int:pk>\ Методы: GET\ Описание: Возвращает экземпляр книги

lab3/book_instance/remove/\<int:pk>\ Методы: GET\ Описание: Удаляет экземпляр книги

lab3/book_instance/update/\<int:pk>\ Методы: GET\ Описание: Обновляет экземпляр книги Данные: 

{
    "code": <string>,
    "quality": <enum ['b', 'g', 'n']>,
    "book": <int> pk of book,
}

lab3/book_instance/create\ Методы: POST\ Описание: Создает экземпляр книги\ Данные: поля модели book_instance

{
    "code": <string>,
    "quality": <enum ['b', 'g', 'n']>,
    "book": <int> pk of book,
    "room": <int> pk of room
}

lab3/book/create\ Методы: POST\ Описание: Создает экземпляр книги Данные: поля модели book

/auth

Как происходит аутентификация?

При помощи /auth/token/login/ получаем токен, затем передаем токен в headers следующим обрзом

Authorization: token <ваш токен>

Входные точки для аутентификации

/auth/users/\ Метод: GET\ Описание: Получить информацию о пользователях

Метод: POST\ Описание: Создает пользователя\ Данные:

{
  'password': ...
  'username':...,
  // Остальные поля, которые есть пользователя
}

/auth/users/me/\ Метод: GET\ Описание: Получить данные текущего пользователя

Метод: PUT\ Описание: Изменение всех полей пользователя\ Данные: необходимо передать все поля, которые есть у пользователя

Метод: PATCH\ Описание: Изменение только некоторых полей пользователя\ Данные: поля, которые хотим изменить

/auth/token/login\ Метод: POST\ Описание: позволяет получить токен для аутентификации\ Данные:

{
  'username': ...,
  'password': ...
}

/auth/token/logout\ Метод: POST\ Описание: удаляет токен
